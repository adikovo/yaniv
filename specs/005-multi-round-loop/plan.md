# Implementation Plan: Multi-Round Loop

**Branch**: `005-multi-round-loop` | **Date**: 2026-06-01 | **Spec**: [spec.md](spec.md)

**Input**: `specs/005-multi-round-loop/spec.md`

## Summary

Replace the round-result modal + `readyForNextRound` flow with a fully automatic loop: after Yaniv the server waits 2 s then deals a new round; the client shows a lightweight "YANIV!" overlay that clears automatically. The existing `RoundResult` modal is repurposed as the end-of-game screen — shows winner + final scores + a "Rematch" button. All players clicking Rematch (or a 10 s timeout) resets scores and starts a fresh game.

## Technical Context

**Language/Version**: Node.js 20 (server), React 19 + Vite (client)

**Primary Dependencies**: Express, Socket.io 4 (server); React Router, Socket.io-client (client)

**Storage**: In-memory (`globals.js` — `games` object)

**Testing**: Jest + socket.io-client (server); no client test suite

**Target Platform**: Local dev / single machine

**Project Type**: Real-time multiplayer web app

**Constraints**: All round state must survive across rounds (scores, eliminated list)

**Scale/Scope**: 2–6 players per room

## Implementation Phases

### Phase A — Server: auto-advance round (US1)

**A1** Write failing tests in `server/tests/multiRound.test.js`:
- T-MR1: yaniv call → `roundEnd` fires → after 2 s both players receive `nextRound` + new `hand` events
- T-MR2: after auto-advance, game state is fully reset (new deck, new top card)

**A2** Refactor `socket.js`:
- Delete `readyPlayers`, `readyTimers`, `readyForNextRound` handler
- In `yaniv` handler, after emitting `roundEnd`:
  - If ≥ 2 active players remain → `setTimeout(() => dealNewRound(room, "nextRound"), 2000)`
  - If only 1 remains → emit `gameOver` with `{ winner: { id, name }, players }`

**A3** Confirm tests green.

---

### Phase B — Server: rematch (US2 end-of-game)

**B1** Write failing test:
- T-MR3: after `gameOver`, both players emit `rematchReady` → `start` event fires, scores reset, eliminated list cleared

**B2** Add `rematchReady` handler in `socket.js`:
- Same ready-set + 10 s timeout pattern as old `readyForNextRound`
- On resolve: reset `game.eliminated = []`, reset all player scores to 0, call `dealNewRound(room, "start")`

**B3** Confirm test green.

---

### Phase C — Client: YANIV overlay + game-over/rematch screen (US1 + US2)

**C1** `game/index.jsx`:
- On `roundEnd`: store in `yanivResult` state for overlay; clear on `nextRound`
- Replace bare `gameOver` boolean with `gameOverData`; show `<RoundResult isGameOver>` when set
- Remove debug button

**C2** New `components/yaniv-overlay/index.jsx`:
- Non-blocking overlay: "YANIV!" + caller name + round scores
- Auto-dismisses on `nextRound` event (or 3 s fallback)

**C3** Update `components/round-result/index.jsx`:
- Add `isGameOver` prop: when true → heading = "Game Over — {winner.name} wins!"; button = "Rematch" → emits `rematchReady`; includes 10 s countdown
- When false (current behaviour): unchanged (not shown during rounds anymore)

---

### Phase D — Spectator mode (US3, P2)

**D1** Server — when a player is eliminated, keep their socket in the room. Add a `spectatorJoin` handler that records the player in `game.spectators` (so they receive `nextRound`/`turn` but no `hand`).

**D2** Server — exclude spectators from `game.players` (already the case after elimination) and ensure `dealNewRound` does not emit a `hand` to spectator sockets.

**D3** Client — after the YANIV overlay clears, if the local player is in `roundEnd.eliminated`, show a Leave / Watch dialog (`components/spectator-prompt/`).

**D4** Client — Watch → render a read-only board (no hand, no action buttons) with an always-visible Exit button → navigates home. Leave → navigate home immediately.
