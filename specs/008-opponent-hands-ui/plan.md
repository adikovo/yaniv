# Implementation Plan: Opponent Hands UI

**Branch**: `008-opponent-hands-ui` | **Date**: 2026-06-09 | **Spec**: [spec.md](spec.md)

## Summary

Add opponent hand areas to the game board. Each opponent shows their name, the correct number of face-down card backs, and their current game score. The layout adapts to player count (2 → top only; 3 → left + right; 4 → left + top + right). The active player's area is highlighted. Disconnected players are removed and the layout re-adjusts.

## Phase 1: Server — Extend `turn` Event with Hand Sizes

### Change: `server/socket.js`

In both `cardFromDeck` and `cardFromTop` turn handlers, after calling the game logic functions and before emitting `turn`, compute `hand_sizes`:

```js
const hand_sizes = {};
for (const id in games[room].players) {
    hand_sizes[id] = games[room].players[id].hand.length;
}
io.to(room).emit("turn", {
    top_card: game_state.top_card,
    current_turn: game_state.current_turn,
    deck: game_state.deck,
    hand_sizes
});
```

Also emit `hand_sizes` in the `start` and `nextRound` events from `dealNewRound`, so clients have initial hand counts when a round begins.

## Phase 2: Client — Game Context Updates

### Change: `client/src/context/game-context.jsx`

1. Add `handSizes` state: `useState({})` — map of `{ [playerId]: count }`
2. Add `opponentScores` state: `useState({})` — map of `{ [playerId]: score }`

3. In the `turn` socket handler, also call `setHandSizes(hand_sizes)` when `hand_sizes` is present in the event.

4. In the `start` socket handler, also call `setHandSizes(hand_sizes)`.

5. Add a `roundEnd` socket handler (or extend the one in game/index.jsx) to extract scores:
   ```js
   socket.on("roundEnd", ({ players }) => {
       const scores = {};
       for (const id in players) scores[id] = players[id].score;
       setOpponentScores(scores);
   });
   ```

6. In the `playerDisconnected` handler, also update `players` state to remove the disconnected player:
   ```js
   setPlayers(prev => prev.filter(p => p.id !== id));
   ```
   (Currently `playerDisconnected` only shows a notice — it does not remove the player from state.)

7. Expose `handSizes`, `setHandSizes`, `opponentScores`, `setOpponentScores` in context value.

## Phase 3: Card Component — Face-Down Support

### Change: `client/src/components/card/index.jsx`

Add a `faceDown` boolean prop. When `faceDown` is true, render `cards/back.png` instead of computing the face-up image path:

```jsx
const src = faceDown ? 'cards/back.png' : getCardImageName(card);
```

The `onClick` handler should be a no-op when `faceDown` (opponent cards are never interactive).

## Phase 4: New `OpponentArea` Component

### New file: `client/src/components/opponent-area/index.jsx`

Props: `{ name, handCount, score, isActive, position }`

Renders:
- A container div with CSS class based on `position` (`opponent-top`, `opponent-left`, `opponent-right`)
- An active highlight class when `isActive` is true
- A row of `handCount` face-down `<Card faceDown />` components, overlapping (fan spread)
- The player's name (truncated via CSS `text-overflow: ellipsis`)
- A score badge (circular, matching existing UI style)

```jsx
export const OpponentArea = ({ name, handCount, score, isActive, position }) => (
    <div className={`opponent-area opponent-${position} ${isActive ? 'active-turn' : ''}`}>
        <div className="opponent-hand">
            {Array.from({ length: handCount }).map((_, i) => (
                <Card key={i} faceDown />
            ))}
        </div>
        <span className="opponent-name">{name}</span>
        <span className="score-badge">{score}</span>
    </div>
);
```

### New file: `client/src/components/opponent-area/styles.css`

- `.opponent-area`: flex column, centered, relative positioning
- `.opponent-hand .card`: negative `margin-left` to create fan overlap (e.g., `-20px` after the first)
- `.opponent-left .opponent-hand`, `.opponent-right .opponent-hand`: same upright fan (no rotation)
- `.active-turn`: add a glow/border highlight (e.g., `box-shadow: 0 0 12px #ffd700`)
- `.opponent-name`: `max-width`, `overflow: hidden`, `text-overflow: ellipsis`, `white-space: nowrap`
- `.score-badge`: circular badge matching the existing style in the game screenshots

## Phase 5: Position Helper

### New file: `client/src/utils/opponent-positions.js`

Pure function `getOpponentPositions(players, localPlayerId)` returns `{ [playerId]: 'top' | 'left' | 'right' }`.

Logic (see data-model.md for full derivation):
- Finds the local player's index in the `players` array
- For 2 players: the other player → `'top'`
- For 3 players: `(localIdx+1)%3` → `'right'`, `(localIdx+2)%3` → `'left'`
- For 4 players: `(localIdx+1)%4` → `'right'`, `(localIdx+2)%4` → `'top'`, `(localIdx+3)%4` → `'left'`

## Phase 6: Game Page — Layout and Opponent Rendering

### Change: `client/src/pages/game/index.jsx`

1. Import `OpponentArea`, `getOpponentPositions`, `handSizes`, `opponentScores` from context.
2. Derive `positionMap = getOpponentPositions(players, player.id)` — memoized with `useMemo`.
3. Replace the existing `<ul>` player list and text turn header with `OpponentArea` components:
   ```jsx
   {players
       .filter(p => p.id !== player.id)
       .map(p => (
           <OpponentArea
               key={p.id}
               name={p.name}
               handCount={handSizes[p.id] ?? 0}
               score={opponentScores[p.id] ?? 0}
               isActive={gameState.current_turn === p.id}
               position={positionMap[p.id]}
           />
       ))
   }
   ```
4. Wrap the existing local player area in a container with `active-turn` class when `gameState.current_turn === player.id`.
5. Apply a CSS Grid layout class on the board container based on `players.length` (or remaining opponent count).

### Change: `client/src/pages/game/styles.css`

Add grid layout:

```css

    grid-template-rows: auto 1fr auto;

    grid-template-areas:
        "left top right"
        "left center right"
        "bottom bottom bottom";
}
```

Each `.opponent-top`, `.opponent-left`, `.opponent-right` maps to the corresponding grid area.
