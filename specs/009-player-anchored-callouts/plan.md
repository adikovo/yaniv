# Implementation Plan: Player-Anchored Yaniv/Asaf Call-Outs

**Branch**: `009-player-anchored-callouts` | **Date**: 2026-06-10 | **Spec**: [spec.md](spec.md)

## Summary

Replace the centered round-end overlay with comic-style "YANIV!" / "ASAF!" call-outs anchored to the acting player's area on every client. "YANIV!" appears over the caller; in an Asaf round, "ASAF!" follows over the countering player a beat later, conveying the +30 penalty. The server adds an explicit `yanivCaller` field to `roundEnd` (replacing the confusing `asafCaller`), so the client anchors call-outs by simple id comparison. Rounds already auto-advance (now after 3s).

## Key Existing Behavior (verified in code)

- `server/socket.js:112` emits `roundEnd` with `{ winner: {id, name}, asaf, asafCaller: {id, name} | null, players, eliminated }`.
  - **Semantics**: `winner` is the round winner. When `asaf` is true, `winner` is the *countering* player and `asafCaller` is the *original Yaniv caller* (who received the +30 penalty). When `asaf` is false, `winner` *is* the Yaniv caller.
  - **This feature makes the caller explicit**: `gameLogic.yanivCall` returns the caller, and `roundEnd` carries `yanivCaller: {id, name}` instead of `asafCaller` (whose only consumer is the overlay being deleted). Client anchoring is then plain id comparison: "YANIV!" on `yanivCaller.id`; "ASAF!" on `winner.id` when `asaf`.
- `server/socket.js:142` auto-deals the next round 3000ms after `roundEnd` (no user interaction).
- `client/src/pages/game/index.jsx` stores the `roundEnd` payload in `yanivResult`, renders `<YanivOverlay>` (centered, fixed) in both the main and spectator views, and clears `yanivResult` 1500ms after `nextRound` — which also drives the eliminated→spectator prompt. That dismissal logic must be preserved.
- Opponents render via `OpponentArea` (`client/src/components/opponent-area/index.jsx`) with a `position` prop (`top`/`left`/`right`); the local player renders in `.local-player-area`. The spectator view renders *all* players as `OpponentArea`s. Anchoring call-outs inside these containers covers every seat, player count, and the spectator case with no coordinate math.

## Phase 1: Server — Explicit `yanivCaller` in `roundEnd`

### Change: `server/gameLogic.js` (+ Jest test, TDD)

`yanivCall(game)` already binds `const caller = getCurrentPlayer(game)`. Return it: `return { winner, asaf, caller }` (drop the `asafCaller: asaf ? caller : null` indirection). Jest test first: asserts the returned `caller` is the calling player in both the normal and asaf cases, and that `winner` is the counterer when asaf.

### Change: `server/socket.js`

In the `roundEnd` emit (~line 112): replace `asafCaller: asafCaller ? {...} : null` with `yanivCaller: { id: caller.id, name: caller.name }`. `asafCaller` is removed from the payload — its only consumer is the old centered overlay, deleted in Phase 6.

## Phase 2: New `CallOut` Component

### New file: `client/src/components/call-out/index.jsx`

Props: `{ variant, penalty }` — `variant: 'yaniv' | 'asaf'`, `penalty: boolean` (show "+30" subtext on the Asaf call-out).

```jsx
import './styles.css';

export const CallOut = ({ variant, penalty }) => (
    <div className={`call-out call-out-${variant}`}>
        <span className="call-out-text">{variant === 'yaniv' ? 'YANIV!' : 'ASAF!'}</span>
        {penalty && <span className="call-out-penalty">+30</span>}
    </div>
);
```

### New file: `client/src/components/call-out/styles.css`

- `.call-out`: `position: absolute`, centered over the parent area (`top: 50%; left: 50%; translate(-50%, -50%)`), `z-index: 60`, `pointer-events: none`.
- `.call-out-text`: large comic-style lettering — bold italic, ~2.2rem (opponents) / scalable, yellow→orange gradient fill (`background-clip: text`) with dark outline via layered `text-shadow` (no `-webkit-text-stroke` dependency alone), slight rotation (−6deg) for the comic feel.
- `.call-out-asaf .call-out-text`: red/orange variant to distinguish from Yaniv.
- `.call-out-penalty`: smaller dark badge under the text ("+30").
- Pop-in animation: `@keyframes call-out-pop` scaling 0 → 1.15 → 1 over ~350ms.
- Must stay within viewport: `max-width: 90vw`, no fixed widths that overflow on the left/right seats.

## Phase 5: Game Page — Anchoring by ID, Sequencing, Removal of Old Overlay

### New hook: `client/src/hooks/use-asaf-sequence.js` (+ test, TDD)

`useAsafSequence(yanivResult)` → `showAsaf: boolean`. `false` initially; when `yanivResult?.asaf` is true, a ~1500ms timer flips it to `true`; resets to `false` when `yanivResult` clears; timer held in a ref and cleaned up on change/unmount. Tested with `renderHook` + fake timers.

### Change: `client/src/pages/game/index.jsx`

1. `const showAsaf = useAsafSequence(yanivResult);`
2. Per-player call-out by plain id comparison against the new explicit payload fields (no derivation helper):
   ```jsx
   const calloutFor = (id) =>
       !yanivResult ? null
       : id === yanivResult.yanivCaller.id ? { variant: 'yaniv', penalty: false }
       : yanivResult.asaf && showAsaf && yanivResult.asafPlayers.some(p => p.id === id) ? { variant: 'asaf', penalty: true }
       : null;
   ```
3. Pass `callout={calloutFor(p.id)}` to every `OpponentArea` (main **and** spectator views) and render `<CallOut>` inside `.local-player-area` when `calloutFor(player.id)` is non-null.
4. Remove both `<YanivOverlay>` renders and the import.

**Timing fit**: `roundEnd` at t=0 → YANIV! pops; ASAF! at t≈1.5s; server `nextRound` at t=3s; `yanivResult` cleared at t=4.5s → call-outs unmount. The eliminated→spectator-prompt logic inside the existing dismissal callback is untouched, satisfying FR-007/FR-008.

**Disconnect edge case**: if the actor disconnected, their `OpponentArea` no longer renders, so the call-out is simply absent — graceful degradation per spec; the round still advances on the server timer.
